# 🚀 Krypton AI Client SDK

Krypton is a powerful, production-grade local LLM gateway that turns any consumer GPU (like an RTX 3050) into a robust, queue-managed private cloud. 

The `krypton-sdk` is the official Python client used to securely interface with a Krypton Gateway. It features automatic queue management, session expirations, and an intelligent **offline fallback** that pings the server owner to wake up their machine if it's asleep!

---

## 📦 Installation

To use the Krypton SDK, install it directly from PyPI:

```bash
pip install krypton-sdk
```

---

## ⚡ Quick Start

You only need two things from the owner of the Krypton Server:
1. Their active **Gateway URL** (e.g., an Ngrok link like `https://their-krypton.ngrok.app`).
2. Your own email address!

### 1. Initialize the Client

```python
from krypton_sdk import KryptonClient

client = KryptonClient(
    email="your.email@example.com", 
    base_url="https://their-krypton.ngrok.app"
)
```

### 2. Request an API Key (Join Queue)

Krypton actively protects its host GPU by strictly managing concurrent requests. Before generating any text, you must request an API key. 

```python
# The client automatically handles the handshake!
client.join_queue()
```

When you call `join_queue()`, **three amazing things** can happen:
1. **Server has room:** You instantly receive an active API key valid for 3 hours. It is saved directly to your client instance, and you're emailed a backup copy.
2. **Server is full:** You are seamlessly added to a secure SQLite Waitlist. When a slot opens, you will be automatically emailed a fresh key.
3. **Server is asleep:** The SDK intercepts the failed connection, silently reaches out to the owner's free Cloud Ping Server, and alerts them via email to boot up their machine.

### 3. Generate Text!

Once you have a key, generating text is incredibly easy. The Krypton SDK handles prompt packaging and header injection natively.

```python
response = client.generate(
    prompt="Explain Quantum Computing in one paragraph.",
    model="llama3",        # Default is llama3
    max_tokens=250,        # Optional token limit
    temperature=0.7        # Optional creativity setting
)

print(response)
```

### 4. Wait, does Krypton stream?

Yes! If you want to build a ChatGPT-like responsive UI, enable `stream=True`. The SDK instantly becomes an async generator.

```python
stream_response = client.generate(
    prompt="Write a poem about GPUs.",
    stream=True
)

print("AI: ", end="", flush=True)
for chunk in stream_response:
    print(chunk, end="", flush=True)
```

## 🔐 Advanced: Using an Existing API Key

Keys generated by the Krypton queue are valid for **exactly 3 hours**. If you close your Python script but your key hasn't expired, you can inject it directly on startup so you don't have to wait in the queue again!

```python
client = KryptonClient(
    email="your.email@example.com", 
    base_url="https://their-krypton.ngrok.app",
    api_key="kr_your_api_key_from_email_here"   # Inject your existing key!
)

# Skip join_queue() and generate instantly!
response = client.generate("Hello again, server!")
```

If your 3 hours are up, Krypton will actively reject your request with a `401 Unauthorized` block to protect the GPU limit. Just call `client.join_queue()` again!

## 🌍 The "Smart Broker" Architecture

Unlike normal libraries that crash if an API is down, `krypton-sdk` acts as a smart broker. 

If the host developer turns off their laptop, `client.generate()` and `client.join_queue()` will gracefully intercept the HTTP crash. It immediately wraps your request up, bounces it securely to the owner's Always-On Render Cloud Beacon, and prints:

`[Krypton] Central GPU Server is offline! Notifying the owner to start it...`

You will get an email the second they turn it on. Happy coding!
